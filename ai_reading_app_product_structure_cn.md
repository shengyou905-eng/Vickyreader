# 知读 — 产品结构方案 (v2)

## 产品定位

> 一个帮助用户建立个人思想系统的 AI 阅读空间。

核心逻辑：

```
阅读 → 标记 → 回顾 → 关联 → 共鸣
```

四个关键词对应四个 Tab：书架（阅读）→ 小U（观察+回顾+关联）→ 随心记（内心书写）→ 明台（公开共鸣）

AI 不做三件事：
- 不代替阅读
- 不频繁弹出
- 不主动骚扰

AI 只做一件事：**让读过的内容不会消失，让思考产生回响。**

---

# 一、整体架构

## 底部导航

```
书架  |  小U  |  随心记  |  明台
```

| Tab | 定位 | 一句话 |
|-----|------|--------|
| 书架 | 输入层 | 用户读书的地方 |
| 小U | 知识中枢 | 读过的内容在这里被理解、串联、回顾 |
| 随心记 | 私人空间 | 完全隔离的书写角落 |
| 明台 | 公共空间 | 选择性公开的阅读共鸣 |

设置（登录、API Key、导出、阅读统计）放在顶栏齿轮图标 ⚙️ 中，不占用底部导航。

## 数据流

```
书架（阅读/标记）
  ↓ 自动收录
小U（整理/关联/洞察）
  ↓ 用户主动选择
明台（公开书籍 + 社交批注）
  ↕ 知识共振
小U 知识图谱（跨书 × 跨人）
```

随心记默认**不在数据流中**——它是独立的私人空间。只有用户在单条记录中主动选择「交给小U思考」，这条记录才会以可撤回引用进入小U的深层回顾与未来思想图谱。

明台在小U之上：用户先有积累，再选择公开。公开后又反过来丰富知识图谱——个人的思考遇到他人的思考。

# 二、书架（输入层）

## 定位

用户进入阅读的入口。目标：**沉浸阅读 + 轻量标记。**

## 阅读页

风格：极简、干净、弱工具感。

选中文字后弹出菜单，**仅三个按钮**：

```
AI解释  |  划线  |  想法
```

不再提供额外操作。用户只负责：

> 注意到了什么 / 想到了什么。

剩下全部自动交给小U。

## 三个按钮的行为

| 按钮 | 行为 |
|------|------|
| AI解释 | 弹出解释面板，支持多轮追问。原文+解释+追问自动收录到小U |
| 划线 | 高亮文本，可选颜色，自动收录到小U |
| 想法 | 弹出输入框，用户写感受。原文+想法自动收录到小U |

每次收录后，轻提示：「已由小U整理」，让用户感知但不必操作。

## 发布到明台（入口）

阅读页顶栏 / 书籍详情页，提供「发布到明台」入口：
- 用户可选择将当前书发布至公共空间
- 发布时决定公开哪些批注
- 需确认版权声明（仅限公版书 / 原创内容 / 已获授权）

## 不做的事

- 不在阅读页弹出 AI 推荐
- 不在阅读页展示关联内容
- 不提供段落 AI 面板（太重）

# 三、小U（知识中枢）⚠️ 第一阶段核心

## 定位

小U不是聊天机器人，而是：

> 基于用户阅读痕迹的长期认知 Agent。

它不是被动存储，而是主动帮用户发现：
- 「我这段时间在关注什么？」
- 「这些内容之间有关联吗？」
- 「我在不同书里对同一个主题有什么不同看法？」

## A+B 人格设计

| | A 人格（长期观察者） | B 人格（阅读嵌入） |
|---|---|---|
| 在哪 | 小U Tab 的引导回顾入口 | 阅读页的 AI解释 |
| 做什么 | 跨书关联、主题洞察、固定问题回顾 | 解释当前段落、回答追问 |
| 优先级 | **第三阶段** | **第一阶段** |

当前阶段先做 B 人格（AI解释），A 人格与知识图谱一起做。

## 阶段一：整理 + 回顾（当前）

### 自动收录（Capture）

零感知。用户划线/想法/AI解释 → 自动进入 `user_entries` 表。

收录内容：
- 原文
- 用户想法 / AI解释 / 追问记录
- 自动标签
- 自动摘要
- 来源书籍 + 章节

多轮追问自动归并到同一条目。

### 智能整理（Organize）

每条进入后自动完成：
1. **打标签**（如：存在主义、认知偏差、写作技巧）
2. **生成一句话摘要**
3. 按标签/主题分组

### 小U页面结构

```
┌──────────────────────────┐
│ 💡 洞察卡片                 │
│ 「你最近关注了 3 个主题」      │
│  疏离感 · 自由意志 · 社会规则  │
├──────────────────────────┤
│ 标签筛选栏                  │
│ 全部 | 哲学 | 心理学 | ...   │
├──────────────────────────┤
│ 知识条目列表                │
│ 《局外人》                  │
│ 「今天，妈妈死了。」          │
│ 🏷 存在主义 文学            │
│ 💬 用户想法：「...」         │
├──────────────────────────┤
│       ✨ 向小U回望           │
│   （固定引导问题入口）         │
└──────────────────────────┘
```

洞察卡片先做静态版本（硬编码），后续动态化。

### 引导回顾（固定模板）

用户点「向小U回望」后，从固定问题中选择：

1. **问答型**：「我最近关注最多的三个主题是什么？」
2. **检索型**：「关于"自由意志"的所有摘录」
3. **摘要型**：「生成本周阅读摘要」

回顾**基于用户自己的阅读条目**回答，不提供自由输入聊天。用户主动授权的随心记只进入深层回顾与未来思想图谱，不混入首页轻量统计。

---

## 阶段三：知识图谱（与 A 人格同步）

> 注：知识图谱是第三阶段。第二阶段是明台。

### 个人知识图谱

- embedding + 向量相似度分析
- 跨书自动关联（「你在《局外人》中也标记过类似观点」）
- 知识图谱可视化：以概念为节点，以关联为边
- 思维导图生成

### 公共知识图谱（明台 + 小U 融合）

- 明台公开批注进入图谱
- 你看到的不是「一本书」，而是「围绕这本书的集体思考」
- 发现：「读这本书的人也在思考这些问题」
- 跨人知识共振——不同读者对同一段话的不同理解

### A 人格（长期阅读观察者）

- 基于用户阅读条目与主动授权片段的长期观察
- 「过去一年我在关心什么？」
- 「帮我写一篇关于 X 主题的阅读综述」
- 年度阅读总结

---

## 用户完整闭环

**不是「整理」，是「回顾」与「共鸣」。**

```
第一天：
打开书 → 划线 → AI解释 → 写想法
（正常阅读，不做任何"整理"动作）

第三天：
打开小U → 洞察卡片告诉你关注的主题
→ 按主题浏览条目，发现自己没意识到的关注方向
（不是在"干活"，是在"回顾自己"）

第七天：
点「向小U回望」 → 「我这一周都在想什么？」
→ 基于本周条目回答
（从自己的阅读痕迹中回望）

第二周：
选择一本书 → 发布到明台 → 公开几段批注
→ 几天后看到有人在你的批注下写了共鸣
（不是在"社交"，是在"遇见同频的人"）

一个月后：
打开某本书 → 翻到某页
→ 小U提示：「你在另一本书里也标记过类似观点」
→ 同时看到明台上还有 3 个人也对这段话有想法
（个人关联 + 公共关联同时浮现）
```

核心：用户只做**读**和**标记**。发现、串联、回顾、共鸣——全是系统和他人带来的。

# 四、明台（公共阅读空间）⚠️ 第二阶段核心

## 命名

**明台**——取「明牌」之意。把底牌亮出来——我的理解、我的困惑、我的共鸣，都摆在这里。

- 明：公开、透明、照亮
- 台：平台、台阁——不是喧闹的广场，是登台论道的台

## 定位

明台不是社交网络，是**书斋**。

> 在图书馆的书页空白处看到前人的铅笔笔记——温热的、安静的、偶遇的。

对比：

| | 社交网络 | 明台 |
|---|---|---|
| 锚点 | 人 | 书 |
| 发现 | 算法推荐 | 以书为媒介的偶遇 |
| 互动 | 点赞、转发、评论 | 共鸣、引用、展开 |
| 氛围 | 喧闹 | 安静 |
| 内容 | UGC 碎片 | 围绕文本的思考 |

## 节奏控制：反 feed 约束

明台不能让用户产生「刷内容」的动作惯性。它的体验应该像：

> 在旧书页边缘，偶然看见别人留下的一句批注。

### 信息密度

- 不做无限下拉流，不做瀑布流，不做全屏滑动卡片
- 首页一屏只露出少量入口：书、主题、页边批注片段
- 批注默认只展示一句核心内容，更多内容由用户主动展开
- 内容以「书 / 段落 / 批注」为锚点，不以「人 / 热度 / 排名」为锚点
- 不展示点赞数、关注数、热榜排名等刺激性指标

### 动效

- 不做自动播放、自动轮播、高频刷新、红点追赶
- 只保留轻微的展开 / 收起 / 淡入动效
- 不使用短视频式左右/上下快速切换
- 批注出现要安静，像页边浮现，不像弹窗打断

### 推荐频率

- 不做连续推荐，不做「看完一个立刻推下一个」
- 「偶遇批注」只在合适的阅读上下文出现，同一阅读会话保持少量
- 明台首页不主动刷新新内容，用户主动进入某本书或某个主题后再深入
- 推荐文案避免「热门、爆款、大家都在看」，使用「有人在这里写下」这类低刺激表达

### 明台验收标准

- 用户打开明台时，没有被催促继续刷的感觉
- 用户看见的是书和批注，不是作者流、热榜流、推荐流
- 停留 30 秒后，页面仍然安静，不主动抢注意力
- 每一次内容出现，都像一次页边偶遇，而不是一次算法投喂

## 核心逻辑

```
用户在书架选择一本书
    │
    ├─ 发布到明台（确认版权声明）
    │     ├─ 书进入明台"馆藏"
    │     └─ 其他人可以「借阅」到自己的书架
    │
    └─ 选择公开的批注
          ├─ 可全选 / 单选 / 不选
          ├─ 公开的批注出现在社交批注层
          └─ 未公开的批注保持私密
```

## 出版权边界

明台仅允许发布：
- 公版书（版权已过期）
- 用户原创内容
- 已获版权方授权的内容

首批预置一批公版经典（论语、孙子兵法、1984 英译等），确保冷启动时有内容可读。

版权声明为必选确认项，发布时弹窗提醒。

## 书斋首页

```
┌──────────────────────────┐
│ 🔍 搜索书籍 / 主题          │
├──────────────────────────┤
│ 📚 最近加入                │
│ 《局外人》·《1984》·《论语》  │
├──────────────────────────┤
│ 🏷 主题分类                │
│ 哲学 · 文学 · 历史 · 科学   │
│ 心理学 · 社会学 · 诗歌      │
├──────────────────────────┤
│ ✎ 页边偶遇                │
│ 「今天，妈妈死了」— 一句批注  │
│ 「己所不欲」— 一段共鸣       │
└──────────────────────────┘
```

## 社交批注层

### 阅读体验

在明台借阅的书中阅读时：
- 侧边 / 底部浮现他人公开批注（标注原作者）
- 点击批注可展开：查看原文上下文、其他共鸣、引用链
- 可发送「共鸣」（不是点赞——共鸣需要写一句话说明为什么）

### 互动方式

| 动作 | 说明 |
|------|------|
| 共鸣 | 写一句话回应此批注（不是无脑点赞） |
| 引用 | 将此批注 + 原文引到自己的小U中 |
| 展开 | 查看此批注的完整讨论链 |

### 批注风格指引

在明台写批注时，提示：

> "像在图书馆的书上写铅笔字那样写。"
>
> ——简短、真诚、面向后来者，而不是算法。

## 冷启动策略

1. **预置公版经典**：选 20-50 本公版书作为首批馆藏
2. **邀请制早期用户**：第一批用户由创始人邀请，质量 > 数量
3. **种子批注**：创始人作为第一个批注者，先耕耘内容

|------|------|
| book_id | 书籍唯一标识 |
| title | 书名 |
| author | 作者 |
| publisher_user_id | 发布者 |
| copyright_status | public_domain / original / authorized |
| cover_url | 封面 |
| category | 分类 |
| published_at | 发布时间 |
| borrow_count | 借阅次数 |

### public_annotations（公开批注表）

| 字段 | 说明 |
|------|------|
| entry_id | 关联 user_entries |
| book_id | 所属书籍 |
| user_id | 批注作者 |
| original_text | 对应原文 |
| annotation_text | 批注内容 |
| position | 原文位置（章节+偏移） |
| resonance_count | 共鸣数 |
| created_at | 时间 |

# 五、随心记（私人空间）

## 定位

整个产品中默认**完全隔绝小U**的地方。

- 默认零 AI 介入
- 不自动分析
- 不自动打标签
- 不自动收录
- 默认不可被小U检索
- 用户可以对单条记录主动选择「交给小U思考」
- 授权采用可撤回引用，不复制为阅读条目
- 授权内容只进入深层回顾与未来思想图谱，不进入小U首页轻量统计
- **不可被明台看到**（绝对不会意外公开）

## 为什么隔绝

如果用户写的每一个字都可能被 AI 分析，他会逐渐失去「我是在为自己写」的感觉。随心记的价值在于：**允许混乱、零散、不被整理。**

## 功能

私人记录。当前支持：
- 文字输入
- 可选标题
- 搜索
- 单条记录主动授权 / 撤回小U读取
- 文本与图片导出

# 八、MVP 分阶段规划

## 第一阶段：小U ⚡ 当前

> 目标：让用户读过的内容可以被回顾、被关联。

### 场景 1：阅读 + 标记 ✅
- EPUB / PDF 阅读
- 划线 / 想法 / AI 解释
- 内容自动存入后端

### 场景 2：小U 整理 🔨 当前
- 小U Tab 基础框架
- 条目按标签分组展示
- 顶部洞察卡片（静态）
- 标签筛选
- 「向小U回望」固定问题入口

### 场景 3：小U 引导回顾 🔨 当前
- 固定回顾问题
- 基于阅读条目回答
- 用户主动授权的随心记可进入深层回顾
- 不提供自由输入聊天

### 场景 4：随心记 ✅
- 私人文本记录
- 默认零 AI 介入
- 单条记录主动授权 / 撤回
- 独立导出

### 交付标准
- 用户可以完整走通「读 → 标记 → 回顾 → 关联」闭环
- 小U 页面结构完整

---

## 第二阶段：明台 🌐

> 目标：让思考产生回响——选择性公开，以书会友。

### 场景 5：发布
- 从书架选择书籍发布到明台
- 选择公开的批注
- 版权声明确认

### 场景 6：书斋浏览
- 明台首页（馆藏、分类、页边偶遇）
- 按主题/年代浏览
- 搜索书籍

### 场景 7：借阅 + 批注阅读
- 借阅明台书籍到自己的书架
- 阅读时看到他人公开批注
- 发送「共鸣」（写一句话回应）
- 引用他人批注到小U

### 场景 8：冷启动
- 预置 20-50 本公版经典
- 邀请制早期用户
- 种子批注内容

### 交付标准
- 用户可以发布书籍、公开批注
- 其他用户可以浏览、借阅、看到批注并互动
- 明台有初始内容，不是空城

## 第三阶段：知识图谱 🧠

> 目标：跨书 × 跨人的知识联结——从「回顾」到「洞察」。

### 场景 9：个人知识图谱
- embedding 向量化所有条目
- 跨书自动关联
- 知识图谱可视化
- 「你在X书中也标记过类似观点」

### 场景 10：公共知识图谱
- 明台公开批注进入图谱
- 跨人知识共振
- 「读这本书的人也在思考这些问题」

### 场景 11：A 人格长期观察
- 基于阅读条目与主动授权片段的长期观察
- 主题综述生成
- 年度阅读总结
- 思维导图

### 交付标准
- 用户能看到自己知识体系的视觉化呈现
- 跨书关联自动浮现
- 公共知识图谱可交互

## 明确砍掉

| 砍掉 | 原因 |
|------|------|
| 「我的」Tab | 无独立内容，合并到设置顶栏 |
| 随心记默认 AI 分析 | 破坏安全感 |
| AI 推荐阅读 | MVP 无数据基础 |
| 段落 AI 面板 | 和阅读页简洁定位冲突 |
| 阅读统计（MVP阶段）| 数据量不够无意义 |
| 明台的关注/粉丝系统 | 明台以书为锚点，不以人为锚点 |
| 明台的 feed 流 | 书斋不是广场，不刷内容 |
| 明台热榜 / 排名 | 会把批注变成竞争，不像书页边缘的偶遇 |
| 短视频式滑动浏览 | 会诱导连续消费，破坏阅读空间的安静感 |
| 高频推荐 / 红点催促 | 明台应该被用户主动进入，不主动追赶用户 |

# 九、设计风格

## 参考

Notion · Readwise · Arc Browser · Apple Books · Kindle

高级 · 简约 · 沉浸 · 克制 · 安静 · 有呼吸感

## 四个 Tab 的视觉气质

| Tab | 气质 |
|-----|------|
| 书架 | 沉静——深色背景，弱化 UI，让书是主角 |
| 小U | 温润——暖色调，卡片感，像翻阅自己的笔记 |
| 随心记 | 素白——极简，像一张白纸 |
| 明台 | 典雅——书卷气，有质感的排版，像古籍阅览室 |
