# AI 阅读 App 产品结构方案

## 产品定位

这不是一个单纯的“AI 阅读器”。

而是：

> 一个帮助用户建立个人思想系统的 AI 阅读空间。

核心逻辑：

```txt
阅读 → 提炼 → 联想 → 沉淀 → 输出
```

AI 的作用不是代替阅读。

而是帮助用户：
- 更快理解
- 更深思考
- 更好沉淀
- 建立长期知识体系

---

# 一、产品整体结构

App 底部采用四个核心 Tab：

```txt
书架
明台
随心记
我的
```

整体结构：

```txt
输入层 → 知识中枢 → 输出层 → 系统层
```

对应关系：

| 板块 | 定位 |
|---|---|
| 书架 | 输入层 |
| 明台 | AI 知识中枢（产品核心） |
| 随心记 | 输出层 |
| 我的 | 系统层 |

---

# 二、书架（输入层）

## 定位

用户进入阅读世界的入口。

目标：
- 沉浸阅读
- AI 理解辅助
- 建立阅读习惯

---

## 页面结构

### 顶部区域

- 搜索
- 最近阅读
- AI 推荐阅读
- 阅读统计入口

---

### 书架区域

书籍卡片展示：

- 封面
- 标题

- 最近阅读时间
- 标签

分类：

- 在读

- 收藏
- 稍后阅读

风格：

- 弱工具感

### 划词功能

支持：

# 三、明台（产品核心）

## 定位

明台不是传统意义上的：

```txt
摘录页
收藏夹
笔记页
```

而是：

> 小U为用户自动生成的思想空间。

这里的核心不是“存储”。

而是：

```txt
收录 → 整理 → 对话
```

---

# 小U（Agent）定位

小U不是聊天机器人。

而是：

> 长期阅读认知 Agent。

它的职责不是单纯回答问题。

而是：

- 收录用户阅读痕迹
- 理解用户长期关注主题
- 建立知识关联
- 生成 Insight
- 形成长期记忆
- 帮助用户沉淀思想

---

# 一、自动收录（Capture）

用户在阅读过程中：

只需要进行：

```txt
AI解释
划线
想法
```

所有内容：

自动进入小U。

用户不需要：

```txt
“手动加入明台”
```

---

## 收录逻辑

### 1. AI解释

```txt
选中 → AI解释
```

小U自动收录：

- 原文
- AI解释
- 自动标签
- 自动摘要
- 追问记录

多轮追问：

自动归并到同一条目。

---

### 2. 划线

```txt
选中 → 划线
```

小U自动收录：

- 原文
- 高亮颜色
- 可选批注
- 自动标签

---

### 3. 想法

```txt
选中 → 想法
```

小U自动收录：

- 原文
- 用户想法
- 自动标签
- 上下文

---

# 自动收录原则

用户：

```txt
感知不到“收录”动作
```

一切自动完成。

但会有轻提示：

```txt
已由小U整理
```

避免用户失去“内容去哪了”的感知。

---

# 二、智能整理（Organize）

每条内容进入系统后：

小U 自动完成：

## 1. 自动打标签

例如：

- 存在主义
- 认知偏差
- 女性主义
- 情绪压抑
- 写作技巧

标签来源：

- 预定义体系
- embedding 相似度
- 用户长期行为

逐渐形成：

```txt
用户个人知识地图
```

---

## 2. 自动摘要

AI 自动生成一句话摘要。

例如：

```txt
作者认为人类的认知偏差并非缺陷，而是进化结果。
```

---

## 3. 自动关联

例如：

```txt
你曾在《局外人》中记录过类似观点。
```

关联：

- 不同书籍
- 不同时间
- 不同主题
- 不同想法

形成长期知识连接。

---

# 三、小U对话（核心差异化）

这是产品真正的差异化部分。

用户不是和普通 AI 聊天。

而是：

> 基于自己的阅读痕迹与思想记录，与小U对话。

---

# 对话类型

## 1. 问答型

例如：

```txt

```

```txt
我在哪些书里记录过“自由意志”？
```

```txt
我最近关注最多的三个主题是什么？
```

---

## 2. 整理型

例如：

```txt
把上周摘录整理成读书周报
```

```txt
把我关于“认知偏差”的内容生成思维导图
```

---

## 3. 推荐型

例如：

```txt
根据我的阅读偏好推荐下一本书
```

```txt
你标记的这段，在另一本书里也有类似观点
```

# 四、随心记（私人空间）

## 定位

随心记是整个产品里：

> 唯一不被 Agent 主动介入的地方。

它是用户的小型私人空间。

这里不强调：
- AI Insight
- 分类
- 分析
- 趋势
- 思维导图

而更强调：

```txt
自由
轻松
安全感
表达欲
```

---

# 随心记的核心逻辑

与“明台”不同。

明台是：

```txt
小U主动理解后的思想空间
```

而随心记是：

```txt
用户保留给自己的原始空间
```

这里允许：
- 混乱
- 零散
- 情绪化
- 不成体系
- 不被整理

这是产品非常重要的人性部分。

---

# 为什么不能让 Agent 深度介入

因为如果：

- 每一句话都被分析
- 每一个情绪都被归类
- 每一次记录都被总结

用户会逐渐失去：

```txt
“我是在为自己写”
```

的感觉。

而会变成：

```txt
“我在给 AI 提供素材”
```

这会破坏安全感。

---

# 正确的 Agent 参与方式

小U在这里：

## 默认安静。

不主动分析。
不主动推送。
不主动生成 Insight。

只有用户主动点击时：

才允许小U参与。

---

# 可选 AI 功能

用户主动触发时：

支持：

- 总结
- 扩写
- 提炼观点
- 整理结构
- 生成标题
- 润色

但：

```txt
默认不介入。
```

---

# 页面气质

- 更柔软

- 更私人
- 更像深夜写东西的地方

知识管理工具

# 阅读行为设计（重要）

用户在阅读过程中：

只需要进行两种动作：

```txt
划线（Highlight）
写想法（Thought）
```

除此之外：

不再提供：

```txt
加入明台
```

这样的主动存放行为。

---

# 为什么要去掉“加入明台”

因为用户会开始纠结：

```txt
这段内容：
到底应该：

放进明台？
还是只划线？
还是写想法？
```

这种“分类决策”会破坏沉浸感。

---

# 正确逻辑

用户只负责：

```txt
注意到了什么
想到了什么
```

剩下全部交给小U。

---

# 小U 自动完成

后台自动：

- 收录
- embedding
- 分类
- 聚合
- Insight 分析
- 主题生成
- 思维导图
- 趋势理解
- 长期记忆
